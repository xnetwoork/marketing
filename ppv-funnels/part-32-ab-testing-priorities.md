# Part 32: A/B Testing Priorities

## Learning Objectives
- Identify the highest-impact A/B tests for PPV funnels
- Sequence tests to avoid conflicting variables
- Use test results to guide funnel iteration during tight timelines

---

## PPV Testing Needs Focus

PPV funnels operate on tight timelines, so testing must be **high-leverage and fast**. Testing too many elements at once creates noise and delays learning.

---

## The PPV Test Priority Stack

1. **Headline / primary promise**
2. **CTA language**
3. **Teaser or hero asset**
4. **Pricing display format**
5. **Guarantee placement**

If you can only run one test, test the headline — it drives all downstream behavior.

---

## Testing Rules for Time-Bound Funnels

- Run tests in short, controlled windows (24–72 hours)
- Avoid multi-variant tests unless traffic is high
- Pause tests 24 hours before event start to reduce risk

Test for learning, not for novelty.

---

## Key Takeaways
- PPV funnels require a tight, prioritized testing roadmap
- Headline and CTA tests produce the largest conversion shifts
- Short test windows match the time-sensitive nature of PPV events
- Stop testing close to event time to protect stability

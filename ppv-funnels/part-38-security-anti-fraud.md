# Part 38: Security & Anti-Fraud

## Learning Objectives
- Protect PPV revenue against fraud and credential sharing
- Balance security measures with user experience
- Reduce chargebacks through proactive fraud prevention

---

## Security Is a Revenue Strategy

PPV events attract fraud, resale, and account sharing. However, overly strict security can frustrate legitimate buyers. The goal is to apply **friction only where risk is highest**.

---

## Core Anti-Fraud Measures

- **Velocity checks** on multiple purchases from one card
- **Geo-IP consistency** to flag abnormal access
- **Device limit policies** for account sharing control
- **Bot and credential stuffing detection** for login endpoints

These measures should run in the background without interrupting most users.

---

## Chargeback Reduction Tactics

- Provide clear event descriptions and pricing on receipts
- Include fast support channels in confirmation emails
- Issue prompt refunds for verified access failures

A fast, fair response reduces chargebacks more than aggressive denial.

---

## Key Takeaways
- Anti-fraud protects PPV margins but must be invisible to legitimate users
- Background checks and device limits are the most effective low-friction controls
- Clear receipts and responsive support reduce chargebacks
- Security measures should be tested before high-traffic events
